Real-time shipment location tracking system

Overview

This API connects Cargowise job creation with Project44 location tracking, providing real-time shipment location updates.

  • Create and manage brokerage jobs
  • Update shipment locations in real-time
  • Query current location by job ID
  • Track shipment status through lifecycle

POST Create Brokerage Job

/webhook/job

Webhook endpoint for Cargowise to notify about new brokerage jobs. Creates a new job and associates it with a shipment.

Request Body
{ "job": "B00001234", // Job ID (B + 8 digits) "shipment": "ABCD12345678", // Shipment ID (4 letters + 8 digits) "status": "ADDED" // Initial status }
Success Response (201 Created)
{ "message": "Job created successfully", "data": { "jobId": "B00001234", "shipmentId": "ABCD12345678", "status": "ADDED", "createdAt": "2025-01-20T10:00:00.000Z", "updatedAt": "2025-01-20T10:00:00.000Z" } }
cURL Example
curl -X POST https://super-alfajores-892212.netlify.app/webhook/job \ -H "Content-Type: application/json" \ -d '{ "job": "B00001234", "shipment": "ABCD12345678", "status": "ADDED" }'

POST Update Shipment Location

/webhook/location

Webhook endpoint for Project44 to send real-time location updates. Updates the location coordinates and optionally the status of a shipment.

Request Body
{ "shipment": "ABCD12345678", // Shipment ID "latitude": "49.0041951", // Latitude coordinate "longitude": "-122.7322901", // Longitude coordinate "status": "IN_TRANSIT" // Optional: Update status }
Success Response (200 OK)
{ "message": "Location updated successfully", "data": { "shipmentId": "ABCD12345678", "jobId": "B00001234", "location": { "latitude": "49.0041951", "longitude": "-122.7322901" }, "status": "IN_TRANSIT", "createdAt": "2025-01-20T10:00:00.000Z", "updatedAt": "2025-01-20T10:05:00.000Z" } }
cURL Example
curl -X POST https://super-alfajores-892212.netlify.app/webhook/location \ -H "Content-Type: application/json" \ -d '{ "shipment": "ABCD12345678", "latitude": "49.0041951", "longitude": "-122.7322901", "status": "IN_TRANSIT" }'

GET Query Current Location

/location/{jobId}

Query the current location and status of a shipment by its job ID. Returns the latest location if available.

Path Parameters
jobId: B00001234 // The job identifier
Success Response (200 OK)
{ "job": "B00001234", "shipment": "ABCD12345678", "status": "IN_TRANSIT", "latitude": "49.0041951", // Only present if location has been set "longitude": "-122.7322901", // Only present if location has been set "createdAt": "2025-01-20T10:00:00.000Z", "updatedAt": "2025-01-20T10:05:00.000Z" }
cURL Example
curl https://super-alfajores-892212.netlify.app/location/B00001234

Response Status Codes

200 OK Request successful (query or update) 201 Created Resource created successfully (new job) 400 Bad Request Invalid request data or validation error 404 Not Found Job or shipment not found 409 Conflict Business rule violation (duplicate job, shipment already associated) 500 Internal Server Error Unexpected server error

Validation Rules

  • Job ID: Must match pattern B followed by 8 digits (e.g., B00001234)
  • Shipment ID: Must match pattern 4 uppercase letters followed by 8 digits (e.g., ABCD12345678)
  • Status: Must be one of: ADDED, IN_TRANSIT, DELIVERED, CANCELLED
  • Latitude: Must be between -90 and 90
  • Longitude: Must be between -180 and 180
  • Business Rules:
    • Job IDs must be unique
    • A shipment can only be associated with one job
    • DELIVERED and CANCELLED shipments cannot change status
    • Location updates less than ~11 meters are ignored
